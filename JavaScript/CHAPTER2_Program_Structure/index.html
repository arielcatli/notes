Chapter 2. Program Structure

Expression

A fragment of code that produces a value is called an expression.

Statement

A statement is a complete command in JavaScript.

Program

A program is a list of statements.

When a statement changes something in the environment of the program, that is called side-effect.

Bindings

A variable that holds values is also called binding.

Declaration/Binding

let caught = 5*5;
The let keyword instructs the program to bind the variable name to the value using the = operator.
The var and const are also binding operators.

Binding Rules

  1. The variable name must start with a non-digit character.
  2. The variable name can only contain the special characters $ and _.
  3. No other special characters should be included and the variable name must not be a keyword.

Environment

The collection of bindings and their values that exists at a given time is called an environment.

When a program starts, the environment is not empty. It always contains bindings that are part of the language standard.

Functions

Values can have function as their type.

A function is a piece of program wrapped in a value.

When invoking a function, it can have arguments which are data to be used by the function.

The console.log Function

The console.log function is used to output some value to the console. Format: console.log(value);

Functions can return values.

Control Flow

A JavaScript program has a straight-line control flow.

Conditional Execution

A JavaScript program can have multiple branches of execution.

            if(condition)
            {
                //do something here if condition is true
            }
            else
            {
                //do something here if condition is false
            }

Loops

While and Do-While Loops

A JavaScript program can loop with definite or indefinite times.

            while(condition)
            {
                //do something while condition is true
            }
        
            do
            {
                //do this while condition is true
            }
            while(condition);

For Loops

            for(initializer; condition; update)
            {
                //do something here
            }

Breaking Out of Loops

The break; statement breaks out the first loop where it is contained.
The continue; statement just continue to the start of the loop again.

Switch Case

            switch(variable)
            {
                case case1:
                    //do something here
                    break;
                case case2:
                    //do something else here
                    break;
                    .
                    .
                    .
                default:
                    //do the default behavior here
                    break;
            }

Comments

Inline comment: //

Block comment: /* */

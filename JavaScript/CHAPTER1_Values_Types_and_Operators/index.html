Chapter 1. Values, Types, and Operators

Values

In JavaScript, all chunks of information are called values.

Numbers

Values of the number type are numeric Values

13 //calculations for integers are precise
9.81 //calculations for fractional numbers may not be
2.223e08

Arithmetic

Operators

  • + - add
  • - - minus
  • * - multiply
  • / - divide
  • % - modulo

Special Numbers

  • Infinity, -Infinity - represents positive and negative infinities
  • NaN - not a number

Strings

Strings are used to represent texts. They are written by enclosing the contents in quotes.

'Down on the Sea'
"Lie on the ocean"
'Float on the ocean'

Strings are represented as Unicode in JavaScript.

To concatenate strings, use the + operator.

"con"+"ca"+"te"+"nate"

A backtick-quoted string can do certain functions:

`half of 100 is ${100/2}`

Unary and Binary Operators

typeof operator gives the type of the value

Unary operators are operators that only take one operand.

Binary operators are operators that take two operands.

Boolean Values

  • false
  • true

JavaScript goes over the Unicode from smallest to the largest or from left to right.

When evaluating a string with > or <, it is evaluated from left to right.

There is only one value in JavaScript that is not equal to itself: Nan (not a number)

Automatic Type Conversion

When an operator is applied to the “wrong” type of value, JavaScript will quietly convert that value to the type it needs, using a set of rules that often aren’t what you want or expect. This is called type coercion.

Further arithmetic operations on NaN keep producing NaN, so if you find yourself getting one of those in an unexpected place, look for accidental type conversions.

The === and !== differs from == and != in which the operands are not type converted.

Short-circuiting of Logical Operations

The || operator (OR) converts the left hand side to Boolean value. If it is true, then it will return it. Otherwise, it will return the right hand side.

The || operator can be used for default values.

null || "Ariel"

The && operator (AND) works the other way around. If the left hand side is evaluated to false, then it will be returned. Otherwise, the value of the right hand side.

null && "Ariel"
